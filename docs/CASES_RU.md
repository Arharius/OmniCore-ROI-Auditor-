## Кейс 1 — Логистика: ТрансЛогик МСК

### Профиль компании
| Параметр | Значение |
|---|---|
| Отрасль | 3PL-логистика, дистрибуция |
| Местоположение | Москва |
| Размер | 180 корпоративных клиентов, 14 менеджеров продаж |
| Средний контракт | 12 000 € |
| Сделок в месяц | 85 |

### Проблема (до автоматизации)

**Ручной цикл обработки заявки:**
1. Клиент присылает заявку по email (хаотичный формат)
2. Менеджер вручную переносит данные в Excel
3. Excel передаётся в 1С бухгалтеру для создания счёта
4. Согласование тарифа — 3–5 итераций по email
5. Договор готовится в Word, подписывается скан-копией

**Итог:** 18 дней от первого контакта до подписанного договора.  
За это время 28% потенциальных клиентов уходили к конкурентам, предлагавшим ответ за 3–4 дня.

| Метрика | До |
|---|---|
| Ручные часы в месяц | 520 ч |
| Ставка менеджера | 15 €/ч |
| % ошибок в документах | 14% |
| Стоимость одной ошибки | 220 € |
| Объём документов | 1 400/мес |
| Длина цикла сделки | 18 дней |
| P(завершения сделки) | 64% |

**Ошибки включали:** неверные реквизиты (ИНН, КПП), ошибки в тарифной таблице, дублирование счетов, некорректные даты доставки. Каждая ошибка — штраф, переделка, потеря доверия клиента.

### Решение

- **CRM (amoCRM)** с воронкой для входящих заявок — автоматический парсинг email
- **Шаблоны документов** с автозаполнением из CRM (договор, счёт, КП)
- **AI-квалификатор** — автоматическая оценка заявки и первичный расчёт тарифа
- **Электронная подпись** (Контур.Диадок) — устранение бумажного документооборота
- Стоимость внедрения: **28 000 €**

### Результаты

| Метрика | До | После | Изменение |
|---|---|---|---|
| Ручные часы/мес | 520 | 130 | −75% |
| % ошибок | 14% | 1.8% | −87% |
| Цикл сделки | 18 дней | 5 дней | −72% |
| P(завершения) | 64% | 89% | +25 п.п. |

### ROI-расчёт (OmniCore)

| Показатель | Значение |
|---|---|
| Экономия на труде (год) | 93 600 € |
| Экономия на ошибках (год) | 237 160 € |
| Прирост выручки (ускорение цикла) | 612 000 € |

| Стоимость внедрения | 28 000 € |

| Узкое место (граф) | согласование тарифа |

## Кейс 2 — Агентство: MOKO Digital

### Профиль компании
| Параметр | Значение |
|---|---|
| Отрасль | Performance-маркетинг |
| Местоположение | Москва |
| Размер | 30 сотрудников, 18 активных клиентских аккаунтов |
| Средний аккаунт | 8 500 €/мес |
| Сделок в месяц | 18 |

### Проблема (до автоматизации)

**Ручной цикл подготовки отчёта:**
1. Менеджер заходит в FB Ads → копирует данные в Excel
2. То же самое с Google Ads, myTarget, ВКонтакте
3. Сведение в общую таблицу (2–3 часа)
4. Форматирование в Google Slides / PowerPoint (1 час)
5. Отправка клиенту, ответы на вопросы по расхождениям

**Итог:** 5 ч/нед × 18 менеджеров = **320 ч/мес** — только на копирование данных.  
Это 8 000 € в месяц зарплаты менеджеров, не производящих ценность.

Дополнительная проблема: 8.5% отчётов содержали цифровые расхождения (разные временные зоны, несовпадение атрибуции). Клиенты получали «неправильные» цифры и запрашивали скидки или дополнительные аудиты.

| Метрика | До |
|---|---|
| Ручные часы в месяц | 320 ч |
| Ставка | 25 €/ч |
| % ошибок в отчётах | 8.5% |
| Стоимость одной ошибки | 150 € |
| Объём отчётов | 280/мес |
| Цикл продажи нового клиента | 21 день |
| P(закрытия сделки) | 71% |

### Решение

- **BI-коннекторы** (Supermetrics / Google Looker Studio Pro): автоматическая выгрузка из 4 рекламных кабинетов
- **Дашборд-шаблоны** с white-label для клиентов — данные обновляются ежедневно
- **Авто-рассылка** резюме недели каждый понедельник в 08:00 без участия менеджера
- **CRM-автоматизация** для входящих лидов: квалификация → КП → договор
- Стоимость внедрения: **15 000 €**

### Результаты

| Метрика | До | После | Изменение |
|---|---|---|---|
| Ручные часы/мес | 320 | 48 | −85% |
| % ошибок | 8.5% | 0.9% | −89% |
| Время подготовки отчёта | 3 дня | 0.3 дня | −90% |
| Цикл продажи | 21 день | 7 дней | −67% |
| P(закрытия) | 71% | 92% | +21 п.п. |

|---|---|

## Кейс 3 — Ритейл: МегаМаркет d.o.o.

### Профиль компании
| Параметр | Значение |
|---|---|
| Отрасль | Розничная торговля, B2B-закупки |
| Местоположение | Белград + Нови-Сад, Сербия |
| Размер | 12 магазинов, закупочный отдел 4 человека |
| Средний B2B-заказ | 1 800 € |
| Заказов в месяц | 45 |

### Проблема (до автоматизации)

**Ручной цикл закупки:**
1. Магазины присылают заявки по Viber/WhatsApp или в общий чат
2. Закупщик сводит заявки вручную в Google Sheets
3. Отправляет заказы поставщикам по email
4. Получает накладные в PDF/скан, разносит вручную в Pantheon ERP
5. Проверка наличия на складе — звонок кладовщику

**Итог:** 8 дней от заявки магазина до поступления товара на полку.  
Магазины компенсировали задержки завышенным страховым запасом (+35% к норме) — замороженный оборотный капитал.

2 200 накладных в месяц × 9.2% ошибок = **202 проблемных документа в месяц**: пересортица, несоответствие цены, неверные единицы измерения. Каждая ошибка — возврат или списание (~95 €).

| Метрика | До |
|---|---|
| Ручные часы в месяц | 380 ч |
| Ставка (Сербия) | 10 €/ч |
| % ошибок в накладных | 9.2% |
| Стоимость одной ошибки | 95 € |
| Объём накладных | 2 200/мес |
| Цикл заказа | 8 дней |
| P(успешного заказа без инцидентов) | 72% |

### Решение

- **EDI-интеграция** с 6 ключевыми поставщиками (стандарт EDIFACT) — заказы отправляются автоматически
- **Авто-импорт накладных** из PDF в Pantheon ERP через OCR
- **Telegram-бот для магазинов** — структурированные заявки вместо чатов
- **Авто-пополнение** по минимальным остаткам для ABC-товаров
- Стоимость внедрения: **19 000 €**

### Результаты

| Метрика | До | После | Изменение |
|---|---|---|---|
| Ручные часы/мес | 380 | 121 | −68% |
| % ошибок | 9.2% | 1.5% | −84% |
| Цикл заказа | 8 дней | 3 дня | −63% |
| Страховой запас | +35% | +12% | −23 п.п. |
| P(успешного заказа) | 72% | 93% | +21 п.п. |
